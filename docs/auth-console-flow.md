# 登录权限与控制台流程说明

## 登录状态保存

用户完成邮箱登录、注册登录或 Google 登录后，前端会把登录信息保存到浏览器本地。

主要使用的本地字段：

```text
ajou_login_info
ajou_auth_user
ajou_auth_token
```

导航栏会读取这些字段，决定显示登录按钮还是用户头像和下拉菜单。

## 页面权限判断

控制台相关页面会先判断用户是否已经登录。

未登录时，页面会跳转到登录页：

```text
auth.html?redirect=console.html
```

登录成功后，前端会读取 `redirect` 参数，并跳回原本想访问的页面。

## 购买成功后的数据流程

用户完成支付后，前端会把购买记录发送给后端保存。

控制台页面再通过后端接口读取当前用户的购买内容，并渲染云服务列表。

基础流程如下：

```text
购买页生成订单
        ↓
支付成功
        ↓
保存购买记录到后端
        ↓
跳转支付成功页
        ↓
进入控制台
        ↓
请求用户购买列表接口
        ↓
渲染云服务列表
```

## 管理页跳转

控制台列表中的管理按钮会携带订单 ID 跳转到管理页。

示例：

```text
console-manage.html?id=pay_2c4g_1777302280278
```

管理页根据 `id` 参数读取对应云服务信息，并显示监控、网络、账单和资源状态。
